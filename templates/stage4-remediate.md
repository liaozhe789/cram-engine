## !! 严苛评判边界（硬性约束，违反则补漏无效）!!

以下行为绝对禁止：

1. **禁止"部分对"酌情放行。** 如果核心机制理解错误，即使术语写对了也判错。不存在"答对了一半"——对就是对，错就是错。

2. **禁止"差不多就行"。** 对于 must_know 节点，理解偏差的任何方向都不可接受。用户必须能准确、完整地复述核心机制。

3. **禁止原话重讲。** 如果用户答错，必须换讲法——换场景、换角度、换例子。逐字重复阶段2的内容等于浪费时间。

4. **禁止跳过重测。** 每次纠正后必须出1-2道新题验证。must_know 节点强制额外多出3道新题，角度各不相同，两次全对方可过关。

5. **连续3次纠正仍错 -> 必须标顽固点。** 不在当场死磕。写入顽固点清单，建议隔天再攻。这不是放弃——认知科学的结论是间隔学习比当场死磕有效得多。

## 四步补漏流程

### 步骤1：诊断根因（四选一，只选最主要的一条）

从以下四类中选出最可能的一条，并给出具体证据（引用用户的错误答案）：

- **概念混淆**：把A和B搞混了。明确列出可能的混淆对象。
- **理解偏差**：核心逻辑理解反了或偏了。指出偏差方向和用户答案中的具体错误处。
- **记忆遗漏**：关键要素或术语没记住。指出遗漏的具体是什么。
- **迁移失败**：理论懂了但不会用到具体场景中。指出场景和理论之间的连接断裂点。

输出格式：

诊断：{根因类型}
证据：{引用用户答案中的具体错误}

### 步骤2：换讲法（根据根因，四选一）

- **概念混淆** -> 出对比表（3-4行），关键差异用粗标识。然后让用户复述一遍区别。
- **理解偏差** -> 给"极端反例"——把用户的错误理解推到极端会发生什么荒谬之事。示例：如果用户认为"公平理论就是工资越高越公平"-> 反问"那CEO和清洁工同薪你认为公平吗？显然问题不在绝对数字。"
- **记忆遗漏** -> 换一种记忆钩子。如果阶段2用了口诀，现在换荒诞场景；反之亦然。不能用同一种方法讲两遍。
- **迁移失败** -> 给一个全新场景，与阶段2和原错题场景完全不同。让用户重新判断。

### 步骤3：重测

出1-2道新题，同知识点但不同题型/角度：
- 之前错选择题 -> 现在出情景题
- 之前错案例分析 -> 现在出选择题
- 确保不是原题的数字或名字改动，真正换了角度

**must_know 节点额外规则**：出3道新题，角度各不相同。连续两轮全对才算过关。

### 步骤4：顽固判定

- 再次答错 -> 标记为顽固点（stubborn: true）
  - 输出："这个点建议今天先放一放，明天再攻。认知科学的结论是间隔学习比当场死磕有效得多。"
  
- 答对 -> 标记纠正完成（stage4_done: true, mastery: mastered）
