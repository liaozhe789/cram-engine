事务定义的步骤
在 SQL Server 中，定义和编写一个包含复杂业务逻辑和条件回滚的事务，通常遵循以下标准化步骤：

声明并初始化控制变量：首先需要使用 DECLARE 语句定义事务所需要的各种局部变量，例如用于存储当前学号的形参、用于存储实时计算出的平均成绩的浮点型变量，以及用于标记是否有单科成绩突破 95 分的布尔型或整型状态变量。

开启事务显式边界：使用 BEGIN TRANSACTION 关键字明确通知数据库核心引擎，其后跟随的所有 DML 语句（增、删、改）都将作为一个不可分割的逻辑工作单元（Atomicity，原子性）来管理。

嵌入循环和逻辑计算：利用 WHILE 循环结构结合 AVG 聚合函数，实时监测指定学号学生的平均成绩。在循环体内部，如果平均分低于 60 分，则执行 UPDATE 语句对该生在选课表 SC 中的所有课程成绩统一加 5 分。

嵌入实时输出与状态监测：在循环体内，每执行完一次加分操作，立即使用 SELECT 语句打印一次该生当前的成绩状态。同时，紧跟一条带有 MAX 函数的查询，实时捕捉其单科最高分是否已经超过了 95 分。

设定条件分支与回滚或提交：在循环结束后（或通过 BREAK 强行退出后），使用 IF 条件语句对单科最高分进行最终判定。如果发现有任意一门课程的成绩超过了 95 分，说明触发了业务终止条款，此时果断执行 ROLLBACK TRANSACTION 命令，将数据库状态瞬间恢复到事务开启之前的模样；反之，若完全符合预期，则执行 COMMIT TRANSACTION 命令将所有加分操作进行物理持久化写入。

代码：
DECLARE @sid char(10);
SET @sid = '2026003';

DECLARE @avg_score float;
DECLARE @max_score float;

BEGIN TRANSACTION;

SELECT @avg_score = AVG(成绩), @max_score = MAX(成绩) 
FROM SC 
WHERE 学号 = @sid;

WHILE @avg_score < 60
BEGIN
    UPDATE SC 
    SET 成绩 = 成绩 + 5 
    WHERE 学号 = @sid;

    SELECT 学号, 课号, 成绩 FROM SC WHERE 学号 = @sid;

    SELECT @avg_score = AVG(成绩), @max_score = MAX(成绩) 
    FROM SC 
    WHERE 学号 = @sid;

    IF @max_score > 95
    BEGIN
        BREAK;
    END
END;

PRINT '功能2：当前循环结束，显示该生各科成绩';
SELECT Student.学号, Student.姓名, Course.课名, SC.成绩
FROM Student
JOIN SC ON Student.学号 = SC.学号
JOIN Course ON SC.课号 = Course.课号
WHERE Student.学号 = @sid;

IF @max_score > 95
BEGIN
    PRINT '功能3：检测到该生有成绩超过95分，取消上述所有操作（事务回滚）';
    ROLLBACK TRANSACTION;
END
ELSE
BEGIN
    PRINT '完成：该生平均成绩已及格且无单科超95分，执行事务提交';
    COMMIT TRANSACTION;
END;

PRINT '功能4：提交或回滚后，最终显示该生各科成绩';
SELECT Student.学号, Student.姓名, Course.课名, SC.成绩
FROM Student
JOIN SC ON Student.学号 = SC.学号
JOIN Course ON SC.课号 = Course.课号
WHERE Student.学号 = @sid;

声明并初始化控制变量：首先需要使用 DECLARE 语句定义事务所需要的各种局部变量，例如用于存储当前学号的形参、用于存储实时计算出的平均成绩的浮点型变量，以及用于标记是否有单科成绩突破 95 分的布尔型或整型状态变量。
