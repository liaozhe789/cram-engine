实验内容与过程
学生（学号，年龄，性别，系名） 
课程（课号，课名，学分，学时）
选课（学号，课号，成绩）
1 根据上面基本表的信息定义视图显示每门课的课名、选课人数、平均成绩 
2 观察基本表数据变化时，视图中数据的变化。
3 利用视图，查询平均成绩最高的学生。

设计器创建视图的步骤：
第一步：打开视图设计器
在 SSMS 左侧的 “对象资源管理器” 中，展开正在使用的数据库（例如 STUDENT）。
找到 “视图” 文件夹，鼠标右键点击它。
在弹出的菜单中选择 “新建视图...” (New View...)。
第二步：添加基础表
此时界面会弹出一个 “添加表” 的小窗口，里面列出了当前数据库中的所有基本表。
鼠标双击（或选中后点击“添加”）需要用到的表。
例如要实现实验五的课程统计：就依次双击添加 Course 表和 SC 表。
添加完毕后，点击 “关闭” 按钮。
第三步：在图形界面中配置多表关系与字段
此时屏幕会分为四个区域（自上而下分别是：关系图区、条件网格区、SQL文本区、结果图形区）。

建立连线： 在最上方的“关系图区”中，用鼠标左键按住 Course 表的 课号 字段，拖动连线到 SC 表的 课号 字段上。
配置左外连接： 鼠标右键点击这条连线选择 “选择‘Course’中的所有行”（或者双击连线，勾选对应的左外连接选项），此时连线会变成带箭头的形状。
勾选输出字段：
在 Course 表中，勾选 课名 前面的复选框。
在 SC 表中，勾选 学号 和 成绩。
第四步：配置聚合与别名（GROUP BY）
在中间的“条件网格区”中，在任意空白处点击右键，选择 “添加分组依据” (Add Group By)。此时网格中会多出一列名为 “分组依据” (Group By)。
调整聚合函数与别名：
课名：其“分组依据”列保持为 Expression（或 Group By）
学号：将其“分组依据”切换为 Count，并在“别名”一列手动打字输入：选课人数。
成绩：将其“分组依据”切换为 Avg，并在“别名”一列手动打字输入：平均成绩。
第五步：保存并生成视图
点击 SSMS 工具栏上的 “保存” 图标（或者直接按下快捷键 Ctrl + S）。
系统会弹出一个小窗口提示输入视图名称，输入 v_Course_Stats，点击确定。


代码：
题目1：
CREATE VIEW v_Course_Stats AS
SELECT Course.课名, COUNT(SC.学号) AS 选课人数, AVG(SC.成绩) AS 平均成绩
FROM Course LEFT JOIN SC ON Course.课号 = SC.课号
GROUP BY Course.课名;

SELECT * FROM v_Course_Stats;



题目2：
UPDATE SC SET 成绩 = 100 WHERE 学号 = '2026001' AND 课号 = 'C01';
SELECT * FROM v_Course_Stats;

题目3：
CREATE VIEW v_Student_Avg AS
SELECT Student.姓名, AVG(SC.成绩) AS 个人平均成绩
FROM Student, SC
WHERE Student.学号 = SC.学号
GROUP BY Student.姓名, Student.学号;

SELECT 姓名, 个人平均成绩
FROM v_Student_Avg
WHERE 个人平均成绩 = (SELECT MAX(个人平均成绩) FROM v_Student_Avg); | 实验内容与过程

第三步：在图形界面中配置多表关系与字段
此时屏幕会分为四个区域（自上而下分别是：关系图区、条件网格区、SQL文本区、结果图形区）。

建立连线： 在最上方的“关系图区”中，用鼠标左键按住 Course 表的 课号 字段，拖动连线到 SC 表的 课号 字段上。
配置左外连接： 鼠标右键点击这条连线选择 “选择‘Course’中的所有行”（或者双击连线，勾选对应的左外连接选项），此时连线会变成带箭头的形状。
勾选输出字段：
在 Course 表中，勾选 课名 前面的复选框。
在 SC 表中，勾选 学号 和 成绩。
第四步：配置聚合与别名（GROUP BY）
在中间的“条件网格区”中，在任意空白处点击右键，选择 “添加分组依据” (Add Group By)。此时网格中会多出一列名为 “分组依据” (Group By)。
调整聚合函数与别名：
课名：其“分组依据”列保持为 Expression（或 Group By）
学号：将其“分组依据”切换为 Count，并在“别名”一列手动打字输入：选课人数。
成绩：将其“分组依据”切换为 Avg，并在“别名”一列手动打字输入：平均成绩。
第五步：保存并生成视图
点击 SSMS 工具栏上的 “保存” 图标（或者直接按下快捷键 Ctrl + S）。
系统会弹出一个小窗口提示输入视图名称，输入 v_Course_Stats，点击确定。


代码：
题目1：
CREATE VIEW v_Course_Stats AS
SELECT Course.课名, COUNT(SC.学号) AS 选课人数, AVG(SC.成绩) AS 平均成绩
FROM Course LEFT JOIN SC ON Course.课号 = SC.课号
GROUP BY Course.课名;

SELECT * FROM v_Course_Stats;



题目2：
UPDATE SC SET 成绩 = 100 WHERE 学号 = '2026001' AND 课号 = 'C01';
SELECT * FROM v_Course_Stats;

题目3：
CREATE VIEW v_Student_Avg AS
SELECT Student.姓名, AVG(SC.成绩) AS 个人平均成绩
FROM Student, SC
WHERE Student.学号 = SC.学号
GROUP BY Student.姓名, Student.学号;

SELECT 姓名, 个人平均成绩
FROM v_Student_Avg
WHERE 个人平均成绩 = (SELECT MAX(个人平均成绩) FROM v_Student_Avg);

为了观察联动，我将基本表 SC 中为“张力”（学号 2026001）的数据库原理成绩，将其从 92 分改为 100 分。

现象：当对基本表 SC 的成绩进行修改后，没有对视图做任何更新操作，但再次查询视图时，视图中的“平均成绩”实时自适应发生了改变。
原因：视图是一个虚拟表，它在数据库中并不物理存储任何行数据，仅仅存储视图的 SQL 定义语句。 当用户查询视图时，数据库管理系统（DBMS）会把视图的定义与外层的查询语句进行“动态组合”，转化为对基本表的直接查询。因此，基本表的数据一旦改变，视图中看到的数据必然是实时最新的。
