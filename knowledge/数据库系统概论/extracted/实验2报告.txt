利用sql建立学生信息数据库，并定义以下基本表：
学生（学号，年龄，性别，系号） 
课程（课号，课名，学分，学时）
选课（学号，课号，成绩）
1定义主码、外码、和年龄、学分、学分的取值范围。
2 在三个表中输入若干记录，注意如果输入违反完整性约束条件的记录系统有何反应。
3 修改数据库日志文件的初始大小；修改一个表，对已有列添加一个约束，并且删除此约束。
-- 创建学生表
CREATE TABLE 学生 (
    学号 char(10) PRIMARY KEY,     
    年龄 int CHECK (年龄 >= 15 AND 年龄 <= 30),
    性别 nchar(1),
    系号 char(5)
);

-- 创建课程表
CREATE TABLE 课程 (
    课号 char(10) PRIMARY KEY,      
    课名 nvarchar(10),
    学分 decimal(3,1) CHECK (学分 > 0), 
    学时 int CHECK (学时 > 0)        
);

-- 创建选课表
CREATE TABLE 选课 (
    学号 char(10),
    课号 char(10),
    成绩 int CHECK (成绩 >= 0 AND 成绩 <= 100),
PRIMARY KEY (学号, 课号),        
FOREIGN KEY (学号) REFERENCES 学生(学号),
    FOREIGN KEY (课号) REFERENCES 课程(课号)  
); | 实验内容与过程

实验结果与分析
违反实体完整性：
操作：INSERT INTO 学生 (学号) VALUES ('2026001');（已有该学号）。
反应：报错“违反了 PRIMARY KEY 约束，不能在对象中插入重复键”。
违反参照完整性:
操作：在选课表中插入一个不存在的学号。
反应：报错“INSERT 语句与 FOREIGN KEY 约束冲突”。
违反用户定义完整性：
操作：将年龄设为 10 或 100。
反应：报错“INSERT 语句与 CHECK 约束冲突”。 | 实验结果与分析
